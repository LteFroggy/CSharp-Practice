# 코딩 연습 - 상속으로 클래스 확장하기

교안의 예제 코드를 직접 따라 작성하고 실행하는 프로젝트입니다.

> 각 과제의 코드를 `Program.cs`에 작성하세요.
> 이전 과제의 코드가 남아있으면 클래스 이름이 겹쳐서 오류가 발생할 수 있습니다. 이전 과제의 코드를 지우거나 주석 처리한 후 다음 과제를 진행하세요.

---

## 1. 상속 구현하기

### 1-1. 기본 상속 문법

`Parent` 클래스에 `Greet()` 메서드를 정의하고, `Child` 클래스가 `Parent`를 상속받아 `Study()` 메서드를 추가하세요. `Child` 인스턴스를 생성하여 두 메서드를 호출합니다.

**실행 결과**

```
안녕하세요!
공부합니다.
```

### 1-2. 동물 클래스 상속

`Animal` 클래스에 `Name` 필드와 `Eat()` 메서드를 정의하고, `Dog`와 `Cat` 클래스가 각각 `Animal`을 상속받아 고유 메서드(`Bark()`, `Meow()`)를 추가하세요. 각 동물 인스턴스를 생성하여 부모 메서드와 자식 메서드를 호출합니다.

**실행 결과**

```
초코이(가) 먹이를 먹습니다.
초코이(가) 멍멍 짖습니다.
나비이(가) 먹이를 먹습니다.
나비이(가) 야옹 웁니다.
```

## 2. Object 클래스

`Product` 클래스에 `Name`과 `Price` 필드를 정의하세요. `Product` 인스턴스를 생성하고 `ToString()`과 `GetType()` 메서드를 호출하여 결과를 출력합니다. 이 메서드들은 모든 클래스가 `Object`에서 상속받은 것입니다.

**실행 결과**

```
Product
Product
```

## 3. 접근 제한자와 상속

`Parent` 클래스에 `private`, `protected`, `public` 필드를 각각 정의하고, `Child` 클래스에서 접근 가능한 필드만 출력하는 `ShowValues()` 메서드를 구현하세요. `Child` 인스턴스를 생성하여 `ShowValues()`를 호출하고, 외부에서 `PublicValue`에 접근합니다.

**실행 결과**

```
2
3
3
```

## 4. base 키워드

`Employee` 클래스에 `protected` 필드 `_department`와 `ShowDepartment()` 메서드를 정의하세요. `Manager` 클래스가 `Employee`를 상속받아 `ShowInfo()` 메서드에서 `base` 키워드로 부모의 필드와 메서드에 접근합니다.

**실행 결과**

```
소속 부서: 개발팀
부서: 개발팀
```

## 5. 생성자와 상속

### 5-1. 부모 생성자 호출 (base)

`Person` 클래스에 `Name` 필드와 매개변수가 있는 생성자를 정의하세요. `Student` 클래스가 `Person`을 상속받아 `Grade` 필드를 추가하고, `base(name)`으로 부모 생성자를 호출합니다.

**실행 결과**

```
Person 생성자 호출: 홍길동
Student 생성자 호출: 3학년
이름: 홍길동, 학년: 3
```

### 5-2. 암시적 기본 생성자 호출

`Animal` 클래스에 기본 생성자를 정의하고, `Dog` 클래스가 `Animal`을 상속받아 자체 생성자를 정의하세요. `base()`를 생략하면 부모의 기본 생성자가 자동으로 호출됨을 확인합니다.

**실행 결과**

```
Animal 기본 생성자
Dog 생성자
```

## 6. 업캐스팅과 다운캐스팅

### 6-1. 업캐스팅

`Animal` 클래스에 `Name` 필드와 `Eat()` 메서드를 정의하고, `Dog` 클래스가 `Animal`을 상속받아 `Bark()` 메서드를 추가하세요. `Dog` 인스턴스를 `Animal` 타입 변수에 할당(업캐스팅)하고, `Eat()`을 호출합니다.

**실행 결과**

```
멍멍이이(가) 먹습니다.
```

### 6-2. 다운캐스팅

6-1의 클래스를 그대로 사용합니다. 업캐스팅된 `Animal` 변수를 `(Dog)`으로 명시적 캐스팅(다운캐스팅)하여 `Bark()` 메서드를 호출하세요.

**실행 결과**

```
멍멍이이(가) 짖습니다.
```

### 6-3. is 연산자로 타입 검사

6-1의 클래스를 그대로 사용합니다. `Animal` 변수가 `Dog` 타입인지 `is` 연산자로 확인한 후 다운캐스팅하여 `Bark()`를 호출하세요.

**실행 결과**

```
멍멍이이(가) 짖습니다.
```

### 6-4. is 패턴 매칭

6-1의 클래스를 그대로 사용합니다. `is` 패턴 매칭을 사용하여 타입 검사와 변환을 동시에 수행하고, `Bark()`를 호출하세요.

**실행 결과**

```
멍멍이이(가) 짖습니다.
```

### 6-5. as 연산자로 안전한 캐스팅

6-1의 클래스에 `Cat` 클래스를 추가하세요. `Cat` 인스턴스를 `Animal`에 업캐스팅한 후, `as Dog`로 캐스팅을 시도하세요. 캐스팅 실패 시 `null`이 반환됨을 확인합니다.

**실행 결과**

```
Dog 타입이 아닙니다.
```

## 10. 멤버 숨기기 (new 키워드)

`Parent` 클래스에 `Value` 필드와 `Show()` 메서드를 정의하세요. `Child` 클래스에서 `new` 키워드로 같은 이름의 멤버를 숨기세요. `Child` 인스턴스를 `Parent` 변수에 할당하여 참조 타입에 따라 다른 멤버가 호출됨을 확인합니다.

**실행 결과**

```
child.Value: 20
parent.Value: 10
Child.Show()
Parent.Show()
```

## 12. 종합 예제: 게임 캐릭터 시스템

`GameCharacter` 추상 클래스에 `Name`, `Health` 속성과 `Attack()` 추상 메서드, `ShowStatus()` 메서드를 정의하세요. `Warrior`와 `Mage` 클래스가 `GameCharacter`를 상속받아 각각의 공격 방식을 구현합니다. `GameCharacter[]` 배열로 캐릭터들을 관리하고 상태와 공격을 출력하세요.

**실행 결과**

```
[아서] 체력: 150
아서이(가) 검으로 25 데미지를 입힙니다!

[멀린] 체력: 80
멀린이(가) 마법으로 40 데미지를 입힙니다!

[란슬롯] 체력: 150
란슬롯이(가) 검으로 30 데미지를 입힙니다!

```
