# 코딩 연습 - 인터페이스

교안의 예제 코드를 직접 따라 작성하고 실행하는 프로젝트입니다.

> 각 과제의 코드를 `Program.cs`에 작성하세요.
> 이전 과제의 코드가 남아있으면 클래스 이름이 겹쳐서 오류가 발생할 수 있습니다. 이전 과제의 코드를 지우거나 주석 처리한 후 다음 과제를 진행하세요.

---

## 1. 인터페이스 구현

### 1-1. 기본 구현

`ICar` 인터페이스에 `Go()` 메서드를 선언하고, `Car` 클래스가 이를 구현하세요. `Car` 인스턴스를 생성하여 `Go()`를 호출합니다.

**실행 결과**

```
자동차가 달립니다.
```

### 1-2. 인터페이스 형식의 변수

`IRepository` 인터페이스에 `Get()` 메서드를 선언하고, `Repository` 클래스가 이를 구현하세요. 인터페이스 타입 변수에 인스턴스를 할당하여 메서드를 호출합니다.

**실행 결과**

```
데이터를 가져옵니다.
```

### 1-3. 구현 클래스의 규칙

`IPerson` 인터페이스에 `Work()`와 `Rest()` 메서드를 선언하세요. `Person` 클래스가 두 메서드를 모두 구현합니다. 인터페이스의 모든 멤버를 반드시 구현해야 함을 확인하세요.

**실행 결과**

```
일을 합니다.
휴식을 취합니다.
```

---

## 2. 생성자 매개변수에 인터페이스 사용

`IBattery` 인터페이스에 `GetName()` 메서드를 선언하세요. `GoodBattery`와 `NormalBattery` 클래스가 각각 구현합니다. `Car` 클래스의 생성자에서 `IBattery`를 매개변수로 받아 저장하고, `Run()` 메서드에서 배터리 이름을 출력하세요.

**실행 결과**

```
고급 배터리를 장착한 자동차가 달립니다.
일반 배터리를 장착한 자동차가 달립니다.
```

---

## 3. 다중 인터페이스 구현

### 3-1. 여러 인터페이스 구현하기

`IAnimal` 인터페이스에 `Eat()`, `IDog` 인터페이스에 `Bark()` 메서드를 선언하세요. `Dog` 클래스가 두 인터페이스를 동시에 구현합니다.

**실행 결과**

```
먹습니다.
짖습니다.
```

### 3-2. 클래스 상속과 인터페이스 구현 함께 사용

`IFlyable` 인터페이스에 `Fly()` 메서드를 선언하고, `Animal` 클래스에 `Breathe()` 메서드를 정의하세요. `Bird` 클래스가 `Animal`을 상속받으면서 `IFlyable`을 구현합니다.

**실행 결과**

```
숨을 쉽니다.
날아갑니다.
```

---

## 4. 명시적 인터페이스 구현

### 4-1. 이름 충돌 해결

`IDog`과 `ICat` 인터페이스에 동일한 이름의 `Eat()` 메서드를 선언하세요. `Pet` 클래스에서 명시적 구현으로 각각 다른 동작을 수행합니다. 인터페이스 타입으로 캐스팅하여 호출하세요.

**실행 결과**

```
개처럼 먹습니다.
고양이처럼 먹습니다.
개처럼 먹습니다.
고양이처럼 먹습니다.
```

### 4-2. 명시적 구현의 특징

`IExample` 인터페이스에 `DoWork()` 메서드를 선언하고, `Example` 클래스에서 명시적으로 구현하세요. 클래스 인스턴스에서는 직접 호출이 불가하고, 인터페이스 타입으로 캐스팅해야 호출 가능함을 확인합니다.

**실행 결과**

```
작업 수행
```

---

## 5. 인터페이스 상속

`IUndoable` 인터페이스에 `Undo()` 메서드를 선언하고, `IRedoable` 인터페이스가 `IUndoable`을 상속받아 `Redo()` 메서드를 추가하세요. `TextEditor` 클래스가 `IRedoable`을 구현하면 두 메서드를 모두 구현해야 합니다.

**실행 결과**

```
실행 취소
다시 실행
```

---

## 6. 인터페이스와 추상 클래스 비교

`IStandard` 인터페이스에 `Run()` 메서드를 선언하세요. `Vehicle` 추상 클래스에 `abstract Back()`과 일반 메서드 `Left()`를 정의합니다. `Car` 클래스가 `Vehicle`을 상속받으면서 `IStandard`를 구현하세요.

**실행 결과**

```
전진
좌회전
후진
```

---

## 7. 게임 개발에서의 인터페이스 활용

### 7-1. 대미지 시스템

`IDamageable` 인터페이스에 `Health` 속성과 `TakeDamage(int damage)` 메서드를 선언하세요. `Player`, `Enemy`, `Building` 클래스가 각각 구현합니다. `Attack(IDamageable target, int damage)` 함수를 만들어 다양한 대상에 대미지를 주세요.

**실행 결과**

```
플레이어가 20 대미지를 받음. 남은 체력: 80
적이 30 대미지를 받음. 남은 체력: 20
건물이 100 대미지를 받음. 남은 내구도: 400
```

### 7-2. 다중 인터페이스로 기능 조합

`IMovable` 인터페이스에 `Move()`, `IAttackable` 인터페이스에 `Attack()` 메서드를 선언하세요. `Hero` 클래스가 두 인터페이스를 모두 구현하고, `Turret` 클래스는 `IAttackable`만 구현합니다.

**실행 결과**

```
영웅이 이동합니다.
영웅이 공격합니다.
포탑이 발사합니다.
```
